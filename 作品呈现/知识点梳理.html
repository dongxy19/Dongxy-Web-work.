page.js

1. ES6 模块导入

- 知识点：使用 `import` 语句导入外部模块（`Navbar` 组件），属于 ES6 模块化规范的基础应用。

2. React 函数组件

- 知识点：通过 `function` 声明定义 React 函数组件（`Hello` 组件），是 React 组件开发的核心方式之一。

3. JSX 语法

- 知识点：在 `return` 中使用 JSX(JavaScript XML)描述组件结构（'div'、'h1' 标签），是 React 中定义 UI 的核心语法。

4. Tailwind CSS 类名使用

- 知识点：通过 `className` 属性为元素添加样式类（`text-indigo-700`、`text-6xl` 等），属于 CSS-in-JS 或原子化 CSS 的实践。

知识点关系与层级

- 基础层:ES6 模块导入（支撑组件间依赖管理）→ 核心层:React 函数组件（定义组件逻辑）→ 表现层:JSX 语法（描述 UI 结构）+ Tailwind CSS(控制样式)。

- 各知识点需协同工作:模块导入确保组件可用,函数组件提供逻辑容器,JSX 定义结构,Tailwind 实现样式。

难易程度

- 均为前端开发基础知识点，难度较低（初级水平）。其中 ES6 模块、React 函数组件、JSX 是 React 开发的入门必备;Tailwind CSS 类名使用需熟悉其原子化命名规则，但上手门槛不高。

Navbar.js

1. React 组件开发

- 函数组件定义：通过 function 声明 Navbar 组件和 RootLayout 组件，是 React 组件开发的基础方式。

- 组件嵌套： RootLayout 中嵌套使用 Navbar 组件，体现 React 组件树的组合特性。

2. JSX 语法

- 描述 UI 结构：使用类 HTML 的 JSX 语法（如 nav 、 div 、 a 标签）定义页面结构，是 React 中渲染 UI 的核心语法。

3. Tailwind CSS 原子化样式

- 类名应用：通过 className 属性添加 bg-slate-800 、 text-white 等 Tailwind 类名，实现快速样式控制。

- 动态类名拼接： RootLayout 中使用模板字符串 ${inter.className} bg-slate-50 拼接动态类名，支持样式灵活组合。

4. ES6 模块导出

- 默认导出：通过 export default 导出 RootLayout 组件，符合 ES6 模块化规范，确保组件可被其他文件导入使用。

知识点关系与层级

- 核心层:React 组件开发（定义功能模块）→ 支撑层:JSX 语法（描述组件结构）+ ES6 模块导出（实现组件复用）→ 表现层:Tailwind CSS(控制视觉样式)。

- 各知识点协同工作:组件提供逻辑容器,JSX 定义结构,ES6 导出实现依赖管理,Tailwind 完成样式渲染。

难易程度

均为前端开发基础知识点，难度较低（初级水平）。其中 React 组件、JSX 是 React 开发入门必备;Tailwind 类名使用需熟悉原子化规则但易上手;ES6 模块导出是现代前端项目的基础规范。
